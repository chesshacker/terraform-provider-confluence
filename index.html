Confluence Provider

The Confluence provider is used to interact with the Cloud Confluence. The provider needs to be configured with the proper credentials before it can be used.

Use the navigation to the left to read about the available data sources.

Example Usage

provider "confluence" {
  site  = "my-site"
  user  = "my-user"
  token = "my-token"
}

resource confluence_content "default" {
  space = "MYSPACE"
  title = "Example Page"
  body  = "<p>This page was built with Terraform<p>"
}

Authentication

Static credentials must be passed to the provider block.

Argument Reference

  • site - (Required) The site is the name of the site and appears in your wiki URL (https://my-site.atlassian.net/wiki/spaces/my-space/). This can also be set via the CONFLUENCE_SITE environment variable.

  • user - (Required) The user is your user's email address. This can also be set via the CONFLUENCE_SITE environment variable.

  • token - (Required) The token is a secret every user can generate. It is similar to a password and should be treated as such. This can also be set via the CONFLUENCE_SITE environment variable. See Manage your account.
